À propos du projet

VeloMax est un système complet de gestion pour les magasins de vélos, développé pour répondre aux besoins spécifiques des commerces spécialisés dans la vente de vélos et de pièces détachées. Le projet a été conçu comme une solution intégrée permettant de gérer efficacement l'inventaire, les commandes, les clients et les fournisseurs.

L'application combine une interface utilisateur intuitive développée en C# avec une base de données MySQL robuste, offrant ainsi une solution complète qui centralise toutes les opérations commerciales d'un magasin de vélos. Elle permet non seulement de gérer les stocks et les ventes, mais aussi de générer des rapports analytiques pour aider à la prise de décision.

Fonctionnalités principales

  • Gestion complète de l'inventaire (vélos, pièces détachées, assemblages)
  • Suivi des fournisseurs et des délais d'approvisionnement
  • Gestion des clients (particuliers et entreprises) avec programmes de fidélité
  • Traitement des commandes avec facturation automatique
  • Système d'alerte pour les stocks bas et les réapprovisionnements nécessaires
  • Tableaux de bord statistiques (ventes, stocks, rentabilité)
  • Exportation des données vers différents formats (XML, JSON, CSV)
  • Interface utilisateur intuitive avec gestion des droits d'accès

-- Création des tables principales

CREATE TABLE Fournisseur (
    siret VARCHAR(14) PRIMARY KEY,
    nom VARCHAR(50) NOT NULL,
    contact VARCHAR(50),
    adresse VARCHAR(100) NOT NULL,
    libelle VARCHAR(50) NOT NULL
);

CREATE TABLE Piece (
    num_piece VARCHAR(10) PRIMARY KEY,
    description VARCHAR(100) NOT NULL,
    date_intro DATE NOT NULL,
    date_disc DATE,
    prix_unitaire DECIMAL(10,2) NOT NULL,
    stock_piece INT NOT NULL DEFAULT 0,
    siret VARCHAR(14),
    FOREIGN KEY (siret) REFERENCES Fournisseur(siret)
);

CREATE TABLE Modele (
    num_modele VARCHAR(10) PRIMARY KEY,
    nom VARCHAR(50) NOT NULL,
    grandeur VARCHAR(10) NOT NULL,
    prix_unitaire DECIMAL(10,2) NOT NULL,
    ligne_produit VARCHAR(50) NOT NULL,
    date_intro DATE NOT NULL,
    date_disc DATE,
    stock_modele INT NOT NULL DEFAULT 0
);

CREATE TABLE Client_Particulier (
    id_client_part INT AUTO_INCREMENT PRIMARY KEY,
    nom VARCHAR(50) NOT NULL,
    prenom VARCHAR(50) NOT NULL,
    adresse VARCHAR(100) NOT NULL,
    telephone VARCHAR(15),
    email VARCHAR(50),
    date_adhesion DATE,
    programme_fidelite VARCHAR(50)
);

CREATE TABLE Client_Entreprise (
    id_client_ent INT AUTO_INCREMENT PRIMARY KEY,
    nom VARCHAR(50) NOT NULL,
    adresse VARCHAR(100) NOT NULL,
    telephone VARCHAR(15),
    email VARCHAR(50),
    nom_contact VARCHAR(50),
    remise FLOAT DEFAULT 0,
    volume_achat INT DEFAULT 0
);

CREATE TABLE Commande (
    num_commande INT AUTO_INCREMENT PRIMARY KEY,
    date_commande DATE NOT NULL,
    adresse_livraison VARCHAR(100) NOT NULL,
    date_livraison DATE,
    id_client_part INT,
    id_client_ent INT,
    FOREIGN KEY (id_client_part) REFERENCES Client_Particulier(id_client_part),
    FOREIGN KEY (id_client_ent) REFERENCES Client_Entreprise(id_client_ent),
    CHECK ((id_client_part IS NULL AND id_client_ent IS NOT NULL) OR 
           (id_client_part IS NOT NULL AND id_client_ent IS NULL))
);

CREATE TABLE Velo (
    num_velo INT AUTO_INCREMENT PRIMARY KEY,
    num_modele VARCHAR(10) NOT NULL,
    FOREIGN KEY (num_modele) REFERENCES Modele(num_modele)
);

CREATE TABLE Composition (
    num_modele VARCHAR(10),
    num_piece VARCHAR(10),
    quantite INT NOT NULL DEFAULT 1,
    PRIMARY KEY (num_modele, num_piece),
    FOREIGN KEY (num_modele) REFERENCES Modele(num_modele),
    FOREIGN KEY (num_piece) REFERENCES Piece(num_piece)
);

CREATE TABLE Ligne_Commande (
    num_commande INT,
    num_velo INT,
    num_piece VARCHAR(10),
    quantite INT NOT NULL DEFAULT 1,
    prix_unitaire DECIMAL(10,2) NOT NULL,
    PRIMARY KEY (num_commande, num_velo, num_piece),
    FOREIGN KEY (num_commande) REFERENCES Commande(num_commande),
    FOREIGN KEY (num_velo) REFERENCES Velo(num_velo),
    FOREIGN KEY (num_piece) REFERENCES Piece(num_piece),
    CHECK ((num_velo IS NULL AND num_piece IS NOT NULL) OR 
           (num_velo IS NOT NULL AND num_piece IS NULL))
);

using System;
using System.Windows.Forms;
using System.Drawing;
using MySql.Data.MySqlClient;

namespace VeloMax
{
    public partial class MainForm : Form
    {
        private MySqlConnection connection;
        private string connectionString;
        
        public MainForm()
        {
            InitializeComponent();
            
            // Configuration de la connexion à la base de données
            connectionString = "Server=localhost;Database=velomax;Uid=root;Pwd=password;";
            connection = new MySqlConnection(connectionString);
            
            // Initialisation des événements
            this.Load += MainForm_Load;
        }
        
        private void MainForm_Load(object sender, EventArgs e)
        {
            try
            {
                // Tester la connexion à la base de données
                connection.Open();
                lblStatus.Text = "Connecté à la base de données";
                lblStatus.ForeColor = Color.Green;
                connection.Close();
                
                // Charger les données initiales
                LoadDashboardData();
            }
            catch (Exception ex)
            {
                lblStatus.Text = "Erreur de connexion: " + ex.Message;
                lblStatus.ForeColor = Color.Red;
                MessageBox.Show("Impossible de se connecter à la base de données. Veuillez vérifier vos paramètres de connexion.",
                                "Erreur de connexion", MessageBoxButtons.OK, MessageBoxIcon.Error);
            }
        }
        
        private void LoadDashboardData()
        {
            // Charger les statistiques pour le tableau de bord
            LoadInventoryStats();
            LoadSalesStats();
            LoadCustomerStats();
            
            // Afficher les alertes de stock bas
            CheckLowStockAlerts();
        }
        
        private void LoadInventoryStats()
        {
            try
            {
                connection.Open();
                
                // Récupérer le nombre total de vélos en stock
                string queryBikes = "SELECT SUM(stock_modele) FROM Modele";
                MySqlCommand cmdBikes = new MySqlCommand(queryBikes, connection);
                object resultBikes = cmdBikes.ExecuteScalar();
                lblTotalBikes.Text = (resultBikes != DBNull.Value) ? resultBikes.ToString() : "0";
                
                // Récupérer le nombre total de pièces en stock
                string queryParts = "SELECT SUM(stock_piece) FROM Piece";
                MySqlCommand cmdParts = new MySqlCommand(queryParts, connection);
                object resultParts = cmdParts.ExecuteScalar();
                lblTotalParts.Text = (resultParts != DBNull.Value) ? resultParts.ToString() : "0";
                
                // Récupérer la valeur totale de l'inventaire
                string queryValue = @"
                    SELECT 
                        SUM(m.prix_unitaire * m.stock_modele) +
                        SUM(p.prix_unitaire * p.stock_piece) AS total_value
                    FROM 
                        Modele m, Piece p";
                MySqlCommand cmdValue = new MySqlCommand(queryValue, connection);
                object resultValue = cmdValue.ExecuteScalar();
                lblInventoryValue.Text = (resultValue != DBNull.Value) 
                    ? string.Format("{0:C}", resultValue) : "0.00 €";
                
                connection.Close();
            }
            catch (Exception ex)
            {
                MessageBox.Show("Erreur lors du chargement des statistiques d'inventaire: " + ex.Message,
                                "Erreur", MessageBoxButtons.OK, MessageBoxIcon.Error);
                connection.Close();
            }
        }
        
        private void LoadSalesStats()
        {
            // Code similaire pour charger les statistiques de vente
        }
        
        private void LoadCustomerStats()
        {
            // Code pour charger les statistiques clients
        }
        
        private void CheckLowStockAlerts()
        {
            try
            {
                connection.Open();
                
                // Trouver les produits avec stock bas (moins de 5 unités)
                string query = @"
                    SELECT 'Vélo' AS type, num_modele AS reference, nom AS description, stock_modele AS stock
                    FROM Modele
                    WHERE stock_modele < 5
                    UNION
                    SELECT 'Pièce' AS type, num_piece AS reference, description, stock_piece AS stock
                    FROM Piece
                    WHERE stock_piece < 5";
                
                MySqlCommand cmd = new MySqlCommand(query, connection);
                MySqlDataAdapter adapter = new MySqlDataAdapter(cmd);
                System.Data.DataTable dt = new System.Data.DataTable();
                adapter.Fill(dt);
                
                // Afficher les alertes dans la grille
                dgvAlerts.DataSource = dt;
                
                // Mettre à jour le compteur d'alertes
                lblAlertCount.Text = dt.Rows.Count.ToString();
                
                connection.Close();
            }
            catch (Exception ex)
            {
                MessageBox.Show("Erreur lors de la vérification des alertes de stock: " + ex.Message,
                                "Erreur", MessageBoxButtons.OK, MessageBoxIcon.Error);
                connection.Close();
            }
        }

            // Ouvrir le formulaire de gestion d'inventaire
            InventoryForm inventoryForm = new InventoryForm(connection);
            inventoryForm.ShowDialog();
            
            // Rafraîchir les données après fermeture
            LoadInventoryStats();
            CheckLowStockAlerts();
        }

Défis techniques

  1. Conception de la base de données

    Créer une structure de base de données qui représente fidèlement les relations complexes entre les différentes entités (vélos, pièces, clients, commandes) tout en garantissant l'intégrité des données et la performance des requêtes.

  2. Gestion des stocks et assemblages

    Développer un système capable de gérer à la fois les stocks de produits finis (vélos) et de composants (pièces détachées), en tenant compte des relations d'assemblage et en maintenant la cohérence des données lors des opérations de vente et d'approvisionnement.

  3. Requêtes et rapports complexes

    Implémenter des requêtes SQL avancées pour générer des rapports analytiques pertinents, comme l'analyse des ventes par période, les prévisions de stock, ou l'identification des produits les plus rentables.

  4. Interface utilisateur intuitive

    Concevoir une interface utilisateur moderne et intuitive qui permette un accès rapide à toutes les fonctionnalités du système, tout en s'adaptant aux différents profils d'utilisateurs (vendeurs, gestionnaires de stock, administrateurs).

Résultats et apprentissages

Le développement de VeloMax m'a permis d'approfondir mes compétences dans plusieurs domaines clés :

  • Conception et optimisation de bases de données relationnelles avec MySQL
  • Développement d'applications Windows Forms en C# avec une architecture en couches
  • Mise en œuvre de requêtes SQL complexes et de procédures stockées
  • Conception d'interfaces utilisateur intuitives et réactives
  • Implémentation de rapports et de visualisations de données
  • Gestion des transactions et de l'intégrité des données
  • Sécurisation des accès à la base de données et gestion des droits utilisateurs

Ce projet m'a également permis de mieux comprendre les enjeux spécifiques de la gestion d'inventaire et de clientèle dans le commerce de détail spécialisé, avec ses contraintes propres liées aux assemblages de produits et à la gestion des pièces détachées.

Fonctionnalités avancées

  1. Analyse prédictive des stocks

    Le système intègre des algorithmes qui analysent l'historique des ventes pour prédire les besoins futurs en stock et générer automatiquement des suggestions de réapprovisionnement, en tenant compte des délais de livraison des fournisseurs.

  2. Gestion des programmes de fidélité

    VeloMax propose une gestion avancée des programmes de fidélité pour les clients particuliers, avec un système de points et d'avantages personnalisés, ainsi qu'un suivi des volumes d'achat pour les entreprises avec application automatique de remises dégressives.

  3. Exportation multi-formats

    L'application permet d'exporter les données (inventaire, commandes, clients) vers différents formats (XML, JSON, CSV) pour faciliter l'intégration avec d'autres systèmes ou l'analyse des données dans des outils externes.

  4. Tableaux de bord dynamiques

    Des tableaux de bord interactifs offrent une vue d'ensemble des performances du magasin (ventes, stocks, rentabilité) avec la possibilité de filtrer les données selon différents critères (période, catégorie de produits, type de client).
